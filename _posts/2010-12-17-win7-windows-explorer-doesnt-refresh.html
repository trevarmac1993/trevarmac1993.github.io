This was a really weird problem. Overnight Windows Explorer on my Windows 7 Enterprise installation stopped refreshing items. For instance, I created a new file and/or renamed an existing one and nothing happened until I manually pressed F5 to refresh. I guess I don't have to mention how annoying this was...

Initially I thought that one of the tools I installed like VirtuaWin, WizMouse or Fences are responsible for that, but apparently they wasn't. Searching for any settings was fruitless so I ended up searching the Windows registry for entries containing "refresh". And look there, I found the following key under HKEY_CLASSES_ROOT\Wow6432Node\CLSID\{BDEADE7F-C265-11D0-BCED-00A0C90AB50F}\Instance


Nice, huh? Don't ask me how that setting came in there. Changing it to "0" (false) fixed the issue and everything worked again just fine. I wonder whether this problem occurs again, 'cause then I have to hunt the prog which causes this setting to be placed.
